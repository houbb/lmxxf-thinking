是什么？

复盘与迭代，这是学习、工作和技能成长中非常核心的自我提升方法。

它不仅是总结，更是形成持续优化闭环的关键环节。

一、什么是复盘与迭代

  • 复盘（Review / Retrospective）：
    对已完成的学习、工作或项目进行系统回顾，分析成功、失败、经验和不足，并提炼出可复用的方法和改进方案。

  • 迭代（Iteration）：
    根据复盘得到的结论，调整策略、方法或行为，并在下一轮实践中应用，以实现持续优化和进步。

简单理解：
复盘 = 回顾 + 总结
迭代 = 调整 + 再实践 → 形成闭环提升

二、复盘的核心目的

  1. 总结经验

    • 提炼成功和失败的原因
    • 明确哪些方法有效、哪些无效

  2. 发现问题与盲点

    • 通过回顾，发现执行过程中未注意到的问题
    • 避免重复犯错

  3. 优化方法和流程

    • 形成标准化或可复用的操作流程
    • 提升下一轮执行效率

  4. 提升自我认知

    • 了解自己的习惯、能力边界和思维模式

三、复盘的方法步骤

  1. 回顾事实

    • 回忆整个学习或工作过程
    • 记录发生了什么、做了哪些动作、取得什么结果

  2. 分析原因

    • 成功原因：哪些方法有效，哪些策略可复制
    • 失败原因：哪些环节不足，导致问题或失误

  3. 提炼经验

    • 形成可总结的原则或方法
    • 用便于记忆的形式记录（如反思日志、笔记卡片）

  4. 规划改进

    • 针对不足提出具体改进措施
    • 明确下一轮的行动计划

四、迭代的核心理念

  1. 小步快跑

    • 每轮实践后做微调 → 避免一次性大改动风险
    • 不断累积改进效果

  2. 反馈驱动

    • 复盘结论 → 指导下一轮实践 → 获取新反馈
    • 保证每次行动都是基于数据和经验

  3. 持续优化

    • 每轮迭代都是学习和成长的机会
    • 随着循环次数增加，方法、技能和效率逐步提升

五、复盘与迭代的工具支持

环节工具功能与落地
复盘记录Notion、Obsidian、纸质日志记录过程、结果、分析、反思
任务与执行追踪Trello、Todoist、ClickUp记录每轮实践任务完成情况，便于回顾
数据分析Excel、Google Sheets、番茄统计 App用数据量化结果，找到瓶颈或模式
知识沉淀Zettelkasten、Obsidian 卡片将复盘经验和总结形成知识网络，支撑下一轮迭代

  1. 简单 6 步复盘流程（可用于学习或工作）：

  2. 事件/任务回顾：今天完成了哪些学习或工作内容？

  3. 结果总结：哪些任务完成得好？哪些未完成？

  4. 成功因素：为什么成功？用哪些方法或策略？

  5. 问题分析：哪些环节出现问题？原因是什么？

  6. 改进措施：下一轮怎么做得更好？

  7. 行动计划：明确具体行动步骤和时间安排

  8. 迭代闭环公式：

实践 → 复盘 → 总结经验 → 调整方法 → 再实践 → 新一轮复盘 → …

七、复盘与迭代的优势

  1. 形成持续改进闭环

    • 学习和工作不再零散，而是不断优化

  2. 提高效率和成果

    • 复用成功经验，规避失败成本

  3. 提升学习深度和反思能力

    • 每轮迭代都会带来对自我和知识更深刻的理解

  4. 支撑长期成长

    • 长期复盘与迭代 → 累积经验 → 形成能力壁垒

💡 一句话总结：

复盘是“回顾与总结”，迭代是“改进与再实践”，两者结合形成闭环，不断优化方法、技能和认知，实现高效学习与持续成长。
