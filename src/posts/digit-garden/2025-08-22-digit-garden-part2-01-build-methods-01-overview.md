# 知识库构建篇的笔记方法

# 一、笔记分类方法

1. 原始笔记（Literature Notes）

   * 来源：书籍、文章、课程、会议。
   * 内容：原始信息、原文摘录、数据和事实。
   * 特点：保留来源，简洁记录，便于后续加工。

2. 理解笔记（Permanent Notes / Zettelkasten）

   * 来源：对原始笔记的思考和总结。
   * 内容：用自己的语言表达理解，提出观点或联系。
   * 特点：独立、清晰、可链接其他笔记。

3. 项目笔记（Project Notes）

   * 来源：工作或实践项目。
   * 内容：任务清单、方案、方案演化记录、问题与解决方案。
   * 特点：实践导向，可操作、可回溯。

4. 灵感笔记（Fleeting Notes）

   * 来源：突发灵感、日常观察、随机想法。
   * 内容：简单记录，随手捕捉，不必完整。
   * 特点：短暂、快速，可后续加工。

---

# 二、笔记方法原则

1. 原子化

   * 每条笔记表达一个核心概念或想法。
   * 便于组合、引用和链接。

2. 双向链接

   * 使用内部链接（如 Obsidian `[[笔记名]]`）连接相关笔记。
   * 形成知识网络，支持非线性思维。

3. 持续迭代

   * 笔记不是一次完成，而是不断完善和扩展。
   * 随时间积累形成知识体系。

4. 可搜索、可复用

   * 添加标签、关键词。
   * 保持文件命名和结构清晰。
   * 方便跨项目或跨主题复用。

5. 分类与结构化

   * 可按主题、领域、时间或项目建立目录。
   * 结合标签（#标签）和数据库（如 Notion/Obsidian Dataview）管理。

---

# 三、笔记工具与辅助方法

* 文本工具：Markdown（轻量、跨平台、可版本控制）
* 笔记平台：

  * 本地工具：Obsidian、Joplin
  * 云端工具：Notion、GitBook
* 辅助方法：

  * Zettelkasten 方法：原子化笔记 + 双向链接
  * Evergreen Notes：长期积累、不断迭代
  * 模板化笔记：统一格式、方便复用

---

# 四、笔记流程示意

1. 捕捉信息 → 原始笔记
2. 消化理解 → 理解笔记
3. 应用实践 → 项目笔记
4. 关联和网络化 → 双向链接 / 标签 / 分类
5. 持续迭代 → 长期积累 / 数字花园

---

💡 核心理念：

> 笔记不仅是信息存储，更是思维网络的节点。通过原子化、双向链接和持续迭代，个人知识库从杂乱信息演化为可探索、可复用的数字花园。
