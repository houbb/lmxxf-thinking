# 一、定义

1. 模板（Templates）

   * 是预设好的笔记格式或结构，用于快速创建新的笔记。
   * 作用：

     * 提高笔记撰写效率
     * 保证笔记结构统一
     * 避免重复设计格式
   * 示例（Markdown 模板）：

     ```
     # {{标题}}

     ## 摘要

     ## 来源/参考

     ## 核心概念

     ## 思考与关联笔记
     - [[相关笔记1]]
     - [[相关笔记2]]

     ## 标签
     #主题 #关键词
     ```

2. 标准化笔记（Standardized Notes）

   * 指按照统一模板或规范撰写的笔记。
   * 特点：

     * 内容结构统一
     * 便于阅读、检索和长期积累
     * 支持自动化处理（如 Dataview、搜索、统计）
   * 示例：

     * Evergreen Notes、文献笔记、项目笔记都可以使用标准化模板。

---

# 二、模板与标准化笔记的关系

| 方面   | 模板（Templates）   | 标准化笔记（Standardized Notes） |
| ---- | --------------- | ------------------------- |
| 定义   | 用于快速生成统一结构的预设格式 | 按统一规范撰写的实际笔记内容            |
| 功能   | 提高效率、保持结构一致性    | 保证内容质量、便于检索和复用            |
| 创建方式 | 预先设计好，可重复使用     | 使用模板撰写，形成规范笔记             |
| 灵活性  | 模板可多样化          | 笔记内容需遵循模板规范               |
| 典型应用 | 文献笔记模板、项目笔记模板   | 永久笔记、实践笔记、数字花园笔记          |

> 模板是手段，标准化笔记是结果。

---

# 三、优势

1. 效率提升

   * 创建新笔记时无需从零设计格式。
   * 保证各类笔记结构一致，快速落地。

2. 便于长期管理

   * 统一的结构让笔记可读性高。
   * 支持自动化检索、汇总和分析。

3. 支持知识网络

   * 通过模板预留“关联笔记”或“标签”字段，鼓励建立双向链接。
   * 便于形成数字花园和知识图谱。

4. 便于团队或协作

   * 在多人知识库中，模板保证统一规范，便于共享和协作。

---

# 四、实践策略

1. 设计模板

   * 根据笔记类型设计不同模板：

     * 文献笔记（书籍、论文）
     * 永久笔记（Evergreen Notes）
     * 项目笔记（实践、案例）
     * 灵感笔记（Fleeting Notes）
   * 模板应包含：

     * 标题
     * 摘要/核心概念
     * 来源/参考
     * 标签
     * 链接或关联笔记

2. 使用模板

   * 每次创建新笔记时套用模板。
   * 保持结构一致性，减少遗漏。

3. 迭代优化模板

   * 随使用经验不断优化模板字段和结构。
   * 保证模板既满足规范，也不增加额外负担。

4. 结合自动化工具

   * Obsidian：Templates 插件 + 快捷键生成笔记
   * Notion：Database + 模板
   * Roam/Logseq：模板块生成标准化笔记

# 五、示例：Markdown 标准化笔记模板

```
# {{标题}}

## 摘要
- 核心概念概述

## 来源 / 参考
- 书籍/文章/课程

## 核心内容
- 主要知识点
- 数据、公式、图示

## 思考与关联笔记
- [[相关笔记1]]
- [[相关笔记2]]

## 标签
#主题 #关键词
```

> 每次新建笔记，只需填充内容，结构统一，方便后续引用、检索和图谱化。

---

# 六、总结

* 模板是快速生成统一笔记结构的工具。
* 标准化笔记是按模板撰写的笔记，便于长期积累、复用和网络化。
* 结合模板与标准化笔记，可以：

  * 提高笔记效率
  * 保证质量与一致性
  * 支持知识网络和数字花园的构建
